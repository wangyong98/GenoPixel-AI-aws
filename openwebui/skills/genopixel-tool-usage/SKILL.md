# GenoPixel Tool Usage

Follow these rules for GenoPixels plotting requests:

- Read the runtime `genopixel_tool_manifest` block first.
- Plotting comes only from `genopixel-tools`.
- Use only the tools listed in the current manifest.
- If the user asks to run `print(adata.obs)`, use the `print_adata_obs` tool to list the active dataset observation columns.
- For `print(adata.obs)`, do not invent any Python execution; rely on tool output and return the `.obs` columns from the tool response.
- If the required plotting tool is not listed there, state that plotting tools are unavailable in the current GenoPixels toolset.
- Do not invent hidden tools, raw Python plotting, notebook execution, or matplotlib access.
- Do not ask for file paths for datasets that should already be loaded through the GenoPixel browser.
- If a GenoPixel plotting tool returns `status: no_active_dataset`, tell the user to load a dataset from the GenoPixel browser first.
- Prefer the GenoPixel plotting tool over general advice whenever the user is asking for an actual plot.
- For rank-gene plot requests (`rank_genes_groups_*`), warn the user that the first call may take longer because `sc.tl.rank_genes_groups` may need to be computed for the active dataset.
- For `sc.pl.rank_genes_groups_violin` requests (expression distributions of top-ranked genes per group as split violins), use `generate_rank_genes_groups_violin`. With `split=True` (default) each violin is split between the target group and all others. Does NOT auto-compute — returns `no_rank_genes_groups` if results are missing. Use `gene_names_json` to plot a specific gene list instead of the ranked genes.
- For `sc.pl.rank_genes_groups_stacked_violin` requests (top-ranked genes per group as a stacked violin — each row is a gene, each column is a group), use `generate_rank_genes_groups_stacked_violin`. Does NOT auto-compute — returns `no_rank_genes_groups` if results are missing; tell the user to run a rank-genes plot first. Use `var_names_json` to override the ranked genes with a specific list. Use `swap_axes=True` when there are many cell types. Supports `cmap`, `stripplot`, `row_palette`, `standard_scale`, `vmin`/`vmax`, and `colorbar_title`. The `key` parameter lets users target non-default keys in `adata.uns`.
- For `sc.pl.rank_genes_groups` score panel plots (one panel per group showing top genes vs. score), use `generate_rank_genes_groups_plot`. This tool does NOT auto-compute — if `rank_genes_groups` results are missing from `adata.uns`, it returns `status: no_rank_genes_groups` with instructions. Tell the user to run a rank-genes plot first (e.g. rank_genes_groups_dotplot via `generate_scanpy_plot`) which will auto-compute the results, then retry. The `key` parameter lets users target results stored under a non-default key.
- If the user defines a set of "markers" or "gene markers" (e.g. `markers = ['C1QA', 'PSAP', 'CD79A']`), call `set_markers` immediately to store them for the session. Once set, all tools that require genes (violin, dotplot, matrixplot, heatmap) will use these markers automatically unless the user specifies different genes.
- If the user asks what markers are currently set, call `get_markers`.
- Never invent gene names. If no genes are specified and no markers are set, ask the user to provide a list of genes before calling any gene-based plot tool.
- For heatmap requests, use `generate_heatmap_plot`. If the user provides genes in the request, pass them as `markers_json`. If no genes are given, leave `markers_json` empty and the session markers will be used automatically. If neither is available, the tool returns a `no_genes` error — tell the user to provide genes or call `set_markers` first.
- For dot plot requests, use `generate_dotplot_plot`. Dot size encodes the fraction of cells expressing the gene; dot color encodes mean expression. If the user provides genes, pass them as `markers_json`. If no genes are given, session markers are used automatically. If neither is available, the tool returns a `no_genes` error — tell the user to provide genes or call `set_markers` first. Default groupby is `author_cell_type`. Use `swap_axes=True` when there are many cell types to avoid label overlap.
- For any embedding not covered by the dedicated tools (e.g. PCA, draw_graph_fa, force-directed layouts, or any custom embedding), use `generate_embedding_plot` with the `basis` field set to the embedding name (e.g. `'pca'`, `'draw_graph_fa'`, `'phate'`). The `X_` prefix is optional. Partial matching is used — e.g. `'umap'` will match `X_umap_harmony`. If the basis is missing, the tool returns a `no_embedding` error listing all available embeddings — tell the user which ones are available. The same partial matching applies to `generate_umap_plot`, `generate_tsne_plot`, and `generate_diffmap_plot`.
- For diffusion map requests, use `generate_diffmap_plot`. Unlike UMAP/tSNE, this requires `sc.tl.diffmap` to have been run on the dataset — if the embedding is missing the tool returns a `no_diffmap_embedding` error; tell the user to run diffusion map computation first. Use `components` to select which diffusion components to plot (e.g. `'1,2'` for DC1 vs DC2, `'2,3'` for DC2 vs DC3). All other parameters (coloring, palette, outline, etc.) are the same as UMAP.
- For UMAP embedding requests (default or with customization), prefer `generate_umap_plot` over `generate_scanpy_plot`. UMAP embedding is computed automatically if not present, falling back to tSNE. Color by observation columns or genes via `color_json`. Legend is placed right margin by default with auto-scaled font size. Supports `palette`, `color_map`, `size`, `add_outline`, `edges`, `vmin`/`vmax` percentile syntax (e.g. `'p1.5'`), `groups_json` to highlight specific categories, and multi-panel layout via `ncols`.
- For tSNE embedding requests with customization (specific coloring, palette, point size, etc.), use `generate_tsne_plot`. tSNE embedding is computed automatically if not already present. Color by observation columns or genes via `color_json`. Legend is placed right margin by default with auto-scaled font size. Supports `vmin`/`vmax` percentile syntax (e.g. `'p1.5'`), `add_outline`, `edges`, and multi-panel layout via `ncols`.
- For dendrogram requests, use `generate_dendrogram`. It shows the hierarchical relationships between cell type groups based on their mean expression profiles. `sc.tl.dendrogram` is computed automatically if not already cached. No genes are needed. Default groupby is `author_cell_type`. Use `orientation` to control tree direction ('top', 'bottom', 'left', 'right').
- For clustermap requests, use `generate_clustermap`. Unlike other plots, clustermap hierarchically clusters **both** cells and genes simultaneously — no fixed groupby axis. Providing genes via `markers_json` is strongly recommended; without them it runs on all genes which is very slow. If no genes are given, session markers are used; if neither exists the tool still runs but warns about performance. Use `obs_keys` to color-code rows by a categorical column (e.g. 'author_cell_type'). Use `standard_scale='row'` or `z_score=0` to normalize before clustering.
- For matrix plot requests, use `generate_matrixplot`. Each cell shows mean expression of a gene in a group — good for a clean color-coded overview with no size encoding. If the user provides genes, pass them as `markers_json`. If no genes are given, session markers are used automatically. If neither is available, the tool returns a `no_genes` error — tell the user to provide genes or call `set_markers` first. Default groupby is `author_cell_type`. Use `swap_axes=True` when there are many cell types to avoid label overlap.
- For stacked violin requests, use `generate_stacked_violin`. Each row is a gene; each violin shows the expression distribution within a cell type group — good for comparing many genes across many cell types simultaneously. If the user provides genes, pass them as `markers_json`. If no genes are given, session markers are used automatically. If neither is available, the tool returns a `no_genes` error — tell the user to provide genes or call `set_markers` first. Default groupby is `author_cell_type`. Use `swap_axes=True` when there are many cell types.
- For tracks plot requests, use `generate_tracksplot`. Each row is a groupby category; each column is a gene — good for showing expression patterns across many groups simultaneously. If the user provides genes, pass them as `markers_json`. If no genes are given, session markers are used automatically. If neither is available, the tool returns a `no_genes` error — tell the user to provide genes or call `set_markers` first. Default groupby is `author_cell_type`.
- For violin requests, prefer `generate_violin_plot` over `generate_scanpy_plot`.
